# 《木构千年》开发计划

> 项目愿景：构建一个专业级中国古代木结构建筑参数化可视化平台

---

## 一、项目现状（MVP v1.0）

### 已完成
- [x] 基础3D场景搭建（Three.js）
- [x] 简化斗栱模型（Box几何体）
- [x] 拆解/组装动画
- [x] 朝代切换（唐/宋/明清）
- [x] 材分等级调整（1-8等）
- [x] 构件显示/隐藏
- [x] 用户玩法指南
- [x] GitHub Pages部署

### 已知问题
- 斗栱模型过于简化（方块代替真实构件）
- 没有真实纹理（纯色材质）
- 缺少完整建筑（只有斗栱，无梁架、屋顶）
- 动画较生硬（无缓动曲线优化）
- 缺少移动端适配
- 构件标签系统不完善

---

## 二、开发阶段规划

### Phase 1: 模型精度提升（2-3周）

**目标**：替换方块模型，使用真实比例的几何体

#### 1.1 斗栱精细化（第1周）
- [ ] 栌斗：带耳、平、欹的复合几何体
- [ ] 华栱：梯形截面+卷杀曲线
- [ ] 横栱（泥道栱、瓜子栱、慢栱）：不同长度的拱
- [ ] 小斗（交互斗、齐心斗、散斗）：区分造型
- [ ] 下昂：斜向构件，带昂嘴曲线

**技术方案**：
- 使用 `THREE.ExtrudeGeometry` 从2D截面拉伸
- 材分制参数驱动尺寸
- 添加木纹法线贴图

#### 1.2 材质系统（第2周）
- [ ] PBR材质（粗糙度、金属度）
- [ ] 程序化生木纹（基于Canvas的纹理生成）
- [ ] 岁月痕迹（污渍、风化效果）
- [ ] 材质切换（新木/旧木/彩绘）

#### 1.3 建筑完整化（第3周）
- [ ] 柱子（含侧脚、生起）
- [ ] 梁架（栿、槫、椽）
- [ ] 屋顶（举折曲线、瓦当）
- [ ] 台基（阶基、栏杆）

**里程碑**：可展示完整的"佛光寺东大殿"单体模型

---

### Phase 2: 数据与算法（2周）

**目标**：建立参数化生成体系

#### 2.1 材分制计算引擎
- [ ] 完整实现《营造法式》材分制度
- [ ] 八等材参数数据库
- [ ] 构件尺寸自动计算
- [ ] 比例验证（防止不合理设计）

#### 2.2 建筑生成器
- [ ] 输入：开间数、进深数、材等
- [ ] 输出：完整建筑3D模型
- [ ] 支持唐、宋、明清不同形制

#### 2.3 数据可视化
- [ ] 尺寸标注系统
- [ ] 材分比例图
- [ ] 结构力流示意
- [ ] 对比视图（唐vs宋vs清）

**里程碑**：输入参数即可生成任意符合法式规范的模型

---

### Phase 3: 交互体验优化（2周）

#### 3.1 动画系统重构
- [ ] 引入GSAP动画库
- [ ] 构件拆解路径规划（避免穿插）
- [ ] 层级拆解（从外到内/从下到上）
- [ ] 组装引导动画

#### 3.2 导航系统
- [ ] 预设视角（正面、侧面、鸟瞰、剖面）
- [ ] 构件聚焦（点击后相机自动对准）
- [ ] 漫游模式（第一人称参观）

#### 3.3 UI/UX改进
- [ ] 加载进度条
- [ ] 操作引导提示
- [ ] 移动端触摸优化
- [ ] 键盘快捷键提示

---

### Phase 4: 内容扩展（3周）

#### 4.1 建筑案例库
- [ ] 佛光寺东大殿（唐）- 已完成基础
- [ ] 南禅寺大殿（唐）
- [ ] 独乐寺观音阁（辽）
- [ ] 晋祠圣母殿（宋）
- [ ] 应县木塔（辽）
- [ ] 故宫太和殿（清）

#### 4.2 知识系统
- [ ] 构件百科（点击显示详细信息）
- [ ] 营造法式原文对照
- [ ] 历史演变时间线
- [ ] 工匠术语词典

#### 4.3 教学模式
- [ ] 引导式拆解（按步骤讲解）
- [ ] 测验模式（识别构件名称）
- [ ] 搭建挑战（按图纸组装）

### Phase 5: 技术深化（2周）

#### 5.1 性能优化
- [ ] LOD（多细节层次）系统
- [ ] 实例化渲染（重复构件）

- [ ] Web Worker（计算密集型任务）

#### 5.2 导出功能
- [ ] 截图（带透明背景）
- [ ] 导出3D模型（GLTF/OBJ）
- [ ] 生成CAD图纸
- [ ] 打印友好布局

#### 5.3 后端服务（可选）
- [ ] 用户保存/分享设计
- [ ] 建筑案例评论
- [ ] 协作编辑

## 三、技术栈规划

### 前端
